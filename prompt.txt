PROJECT OVERVIEW
----------------------------------------------------------------
A single-file, menu-driven C program that renders ASCII shapes
on a 60x20 character canvas.
  Background character : _
  Drawn pixel          : *
Up to 100 shapes can be stored. The canvas is fully redrawn
after every add, delete, or modify operation.

----------------------------------------------------------------
KEY SPECIFICATIONS
----------------------------------------------------------------

  Constants:
    #define W   60      // canvas width
    #define H   20      // canvas height
    #define MAX 100     // max stored objects

  Data Structure:
    typedef struct {
        int type;       // shape identifier (0-3)
        int p[6];       // up to 6 coordinate/parameter values
    } Obj;

  Global Variables:
    char  canvas[H][W];
    Obj   objs[MAX];
    int   count = 0;
    const char *names[] = {"Circle","Rectangle","Line","Triangle"};
    const int   npar[]  = {3, 4, 4, 6};

----------------------------------------------------------------
SUPPORTED SHAPES & PARAMETERS
----------------------------------------------------------------
  Shape         Parameters stored in p[]
  ----------    ----------------------------
  Circle        cx, cy, radius
  Rectangle     x1, y1, x2, y2
  Line          x1, y1, x2, y2
  Triangle      x1, y1, x2, y2, x3, y3

----------------------------------------------------------------
CORE FUNCTIONS
----------------------------------------------------------------

  *** DRAWING ALGORITHMS ***

  draw_line  —  Bresenham's Line Algorithm
  -----------------------------------------
    int dx = abs(x2-x1), dy = abs(y2-y1);
    int sx = x1<x2 ? 1:-1,  sy = y1<y2 ? 1:-1;
    int err = dx-dy, e2;
    while(1) {
        set_pixel(x,y);
        if(x==x2 && y==y2) break;
        e2 = 2*err;
        if(e2 > -dy) { err -= dy; x += sx; }
        if(e2 <  dx) { err += dx; y += sy; }
    }

  draw_circle  —  Midpoint Circle Algorithm (8-octant symmetry)
  ---------------------------------------------------------------
    int x=0, y=r, d=3-2*r;
    while(r>=0 && x<=y) {
        // plot all 8 symmetric points around (cx, cy)
        if(d>=0) { d += 4*(x-y)+10; y--; }
        else       d += 4*x+6;
        x++;
    }

  draw_rectangle  —  4 calls to draw_line (top/bottom/left/right)
  draw_triangle   —  3 calls to draw_line connecting each vertex pair

  *** CANVAS UTILITIES ***

  clear_canvas()           fills canvas[H][W] with '_'
  set_pixel(int x, int y)  bounds-checked; writes '*' to canvas[y][x]
  display_picture()        prints canvas row by row
  redraw()                 clear_canvas() + draw_obj() for all objects

----------------------------------------------------------------
OBJECT MANAGEMENT
----------------------------------------------------------------
  add_object()     reads shape + params -> appends to objs[count++] -> redraw
  delete_object()  reads index -> shifts array left -> count-- -> redraw
  modify_object()  reads index -> overwrites params -> redraw
  list_objects()   prints "i. ShapeName: p0 p1 p2 ..."

----------------------------------------------------------------
MAIN MENU
----------------------------------------------------------------
  1. Add    2. Delete    3. Modify
  4. List   5. Display   6. Clear    7. Exit

  Option 6 : resets count=0 and clears canvas
  Option 7 : exits with return code 0

BUGS FIXED DURING DEVELOPMENT
----------------------------------------------------------------
  Function        Bug                    Fix
  -----------     --------------------   -----------------------------------
  clear_canvas    Wrong loop logic       for r->H, for c->W: canvas[r][c]='_'
  set_pixel       Missing bounds check   if(x>=0 && x<W && y>=0 && y<H)
  draw_line  ***  Incorrect Bresenham    e2=2*err; separate x and y step
                  step logic             checks — FIXED WITH CLAUDE SONNET 4.6

----------------------------------------------------------------
SPECIAL FEATURE: TELEMETRY / DEBUG HOOKS
----------------------------------------------------------------
  Three debug_log() call-points embedded for tracing:

  Hook  Location       What it logs
  ----  -----------    -------------------------------------------
  H1    main loop      Menu choice selected, current object count
  H2    add_object     Confirms add; logs count, type, first param
  H3    read_params    Param read success/failure per shape type
